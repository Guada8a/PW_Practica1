# Práctica #4: Módulos

## Módulos
<p style="text-align:justify">
Un módulo es un conjunto de código y de datos relacionados lógicamente. Encapsula (o encierra)
detalles de la implementación (o sea, de la forma en cómo está escrito el código), de esta forma el
que usa el módulo no necesita saber «cómo está hecho» el módulo, sino solamente «cómo se usa«. Y
como sólo se necesita saber «cómo se usa» un módulo, lo único que se expone al mundo exterior es
la forma de usarlo (tal como sucede con los paquetes que descargas de npmjs.com). Finalmente,
combinando varios módulos se pueden crear aplicaciones mucho más grandes.
</p>

## Javascript Objects
<p style="text-align:justify">
Name/Value pairs : a name which maps to a value

The name may be defined more than once but only can have one value in any given context.

The value may be another name/value pair
Un Javascript Object es : a collection of name value pairs
</p>
¿Qué puede contener un objeto:?

- Datos primitivos
- Funciones (method)
- Otros objetos
### Object literal
Name/value pair separated by colons and surrounded by curly braces
